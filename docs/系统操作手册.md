## 一、系统概述

### 1.1 系统简介

本系统为**员工季度考核管理系统**，用于管理实施交付部和产品研发部的季度绩效考核工作。系统覆盖从数据导入、项目参与度填报、公共积分申报、360 互评到最终成绩计算的完整考核流程。

### 1.2 系统角色

系统定义了四种角色，各角色拥有不同的功能权限：

| 角色 | 说明 |
|------|------|
| **管理员** | 拥有全部系统权限，负责员工/项目管理、周期配置、参数设定、成绩计算与归档 |
| **项目经理** | 负责填报所管项目的参与度，可进行公共积分申报和 360 评价 |
| **普通员工** | 可申报公共积分、完成 360 互评任务、查看个人成绩 |
| **领导** | 负责对公共人员进行工作目标完成度评分、编辑领导评语、查看最终成绩 |

### 1.3 考核类型

| 考核类型 | 说明 |
|----------|------|
| 基层管理人员 | 中层管理岗位 |
| 公共人员 | 公共/支撑岗位 |
| 业务人员 | 业务实施岗位 |
| 产品研发人员 | 产品与研发岗位 |

### 1.4 考核周期阶段

每个考核周期依次经过 **5 个阶段**：

| 阶段 | 名称 | 说明 |
|------|------|------|
| 阶段 1 | 数据导入 | 管理员导入员工、项目数据，配置考核参数 |
| 阶段 2 | 填报申报 | 项目经理填报参与度，员工申报公共积分 |
| 阶段 3 | 在线评分 | 全员进行 360 互评，领导对公共人员评分 |
| 阶段 4 | 结果查看 | 管理员计算最终成绩，各角色查看结果 |
| 阶段 5 | 确认结束 | 管理员确认考核完成，归档数据 |

## 二、系统登录与退出

### 2.1 登录

1. 在浏览器中打开系统地址，进入登录页面。
2. 在 **手机号** 输入框中输入 11 位手机号。
3. 在 **密码** 输入框中输入登录密码。
4. 点击 **「登录」** 按钮。
5. 登录成功后，系统显示"欢迎回来，{您的姓名}"，自动跳转至工作台。

> **提示：** 初次登录的默认密码为手机号后 6 位。如忘记密码，请联系管理员重置。

### 2.2 退出

1. 点击页面右上角的 **用户头像**。
2. 在下拉菜单中点击 **「退出登录」**。
3. 在弹出的确认对话框中点击 **「确定」**。
4. 系统清除登录状态并跳转至登录页面。

## 三、界面布局说明

### 3.1 整体布局

系统采用左侧菜单 + 顶部导航栏 + 主内容区的经典布局：

```
┌──────────────────────────────────────────────────┐
│  顶部导航栏：系统名称 │ 当前周期 │ 主题切换 │ 用户头像 │
├──────────┬───────────────────────────────────────┤
│          │                                       │
│  左侧    │                                       │
│  菜单栏  │            主内容区域                   │
│          │                                       │
│          │                                       │
├──────────┴───────────────────────────────────────┤
│  侧边栏底部：当前用户姓名（角色）                    │
└──────────────────────────────────────────────────┘
```

### 3.2 顶部导航栏

| 元素 | 位置 | 说明 |
|------|------|------|
| 系统名称 | 左侧 | 点击可跳转至工作台首页 |
| 当前周期 | 右侧 | 显示当前活跃考核周期名称及阶段标签（如"第 2 阶段：填报申报"） |
| 主题切换 | 右侧 | 点击切换显示模式：跟随系统 / 浅色模式 / 深色模式 |
| 用户头像 | 右侧 | 点击展开下拉菜单：个人信息、修改密码、退出登录 |

### 3.3 左侧菜单栏

- 菜单栏可折叠/展开。
- 不同角色看到的菜单项不同（详见附录 A）。
- 侧边栏底部展示当前登录用户的姓名和角色。

## 五、基础数据管理（管理员）

> **适用角色：** 仅管理员

### 5.1 员工管理

**菜单路径：** 基础数据 → 员工管理

#### 5.1.1 查看员工列表

页面以表格展示所有员工信息，包含以下列：

| 列名 | 说明 |
|------|------|
| 姓名 | 员工姓名 |
| 手机号 | 员工手机号（登录账号） |
| 部门 | 所属部门 |
| 组别 | 所属组别 |
| 职务 | 员工职务 |
| 职级 | 员工职级 |
| 角色 | 系统角色（管理员/项目经理/普通员工/领导） |
| 考核类型 | 考核归属类型 |
| 状态 | 是否启用 |
| 操作 | 编辑 / 删除 / 重置密码 |

**搜索与筛选：**
- **搜索框：** 按姓名或手机号搜索
- **筛选下拉：** 部门、组别、角色、考核类型
- **分页：** 支持 10 / 20 / 50 / 100 条/页

#### 5.1.2 新增员工

1. 点击页面左上方的 **「+ 新增」** 按钮。
2. 在弹出的对话框中填写以下信息：

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 姓名 | 是 | 员工姓名 |
| 手机号 | 是 | 11 位手机号，同时作为登录账号 |
| 部门 | 是 | 选择所属部门 |
| 组别 | 否 | 所在组别名称 |
| 职务 | 否 | 职务名称 |
| 职级 | 否 | 如 T1-T9、S1-S5、P1-P5 等 |
| 角色 | 是 | 选择系统角色 |
| 考核类型 | 是 | 选择考核归属类型 |
| 副考核类型 | 否 | 适用于双重考核身份的员工 |
| 是否启用 | — | 默认启用 |

3. 点击 **「确定」** 保存。

> **注意：** 新增员工的初始密码为手机号后 6 位。

#### 5.1.3 编辑员工

1. 在员工列表中找到目标员工，点击操作列的 **「编辑」**。
2. 修改需要变更的字段。
3. 点击 **「确定」** 保存。

#### 5.1.4 删除员工

1. 在员工列表中找到目标员工，点击操作列的 **「删除」**。
2. 在弹出的确认框中点击 **「确定」**。

> **注意：** 删除操作不可恢复，请谨慎操作。

#### 5.1.5 重置密码

1. 在员工列表中找到目标员工，点击操作列的 **「重置密码」**。
2. 系统将该员工密码重置为其手机号后 6 位。

#### 5.1.6 Excel 批量导入

1. 点击 **「下载模板」** 获取标准 Excel 模板。
2. 按照模板格式填写员工数据。
3. 点击 **「上传」** 按钮，选择填好的 Excel 文件。
4. 可勾选 **"全量更新"** 选项（将清除原有数据后重新导入）。
5. 系统自动解析并导入数据，导入完成后显示结果。

#### 5.1.7 其他操作

- **「重新加载」**：刷新员工列表数据。

页面以表格展示所有项目信息，包含以下列：

**搜索与筛选：**
- **搜索框：** 按项目编号或项目名称搜索
- **筛选下拉：** 项目类型、部门、项目状态

#### 5.2.2 新增项目

1. 点击 **「+ 新增项目」** 按钮。
2. 在弹出的对话框中填写以下信息：

| 字段 | 是否必填 | 说明 |
|------|----------|------|

| 项目名称 | 是 | 项目名称 |
| 项目类型 | 是 | 下拉选择 |
| 实施方式 | 否 | 服务 / 产品+服务 |
| 部门 | 是 | 所属部门 |

| 自研收入 | 否 | 自主研发收入 |

1. 点击页面上方的 **「签约概率设置」** 按钮，打开右侧抽屉面板。
2. 面板列出所有**未签约**项目，每个项目旁有一个概率输入框（0 ~ 1）。
3. 修改概率值后，点击 **「保存并重算」**。
4. 系统重新计算所有相关项目的经济规模系数。

#### 5.2.5 Excel 导入

操作方式与员工导入类似：
1. 点击 **「下载模板」** 获取模板。
2. 填写项目数据。
3. 点击 **「上传」** 导入。

## 六、考核设置（管理员）

> **适用角色：** 仅管理员

### 6.1 考核周期管理

**菜单路径：** 考核设置 → 考核周期

#### 6.1.1 查看周期列表

表格展示所有考核周期：

| 列名 | 说明 |
|------|------|
| 周期名称 | 如"2026 年 Q1 季度考核" |
| 当前阶段 | 当前所处阶段（1-5） |
| 是否活跃 | 是/否 |
| 是否归档 | 是/否 |
| 操作 | 激活 / 归档 / 上一阶段 / 下一阶段 |

#### 6.1.2 创建新周期

1. 点击 **「+ 创建周期」** 按钮。
2. 输入周期名称（如"2026 年 Q2 季度考核"）。
3. 点击 **「确定」**。

> **注意：** 新建周期会自动继承上一周期的员工名单和考核参数。

#### 6.1.3 激活周期

- 点击目标周期的 **「激活」** 按钮，使其成为当前活跃周期。
- 系统同一时间只能有**一个**活跃周期。

#### 6.1.4 切换阶段

- 点击 **「> 下一阶段」** 按钮推进到下一阶段。
- 点击 **「< 上一阶段」** 按钮回退到上一阶段。

> **注意：** 阶段切换影响全系统功能的可用性，请确认准备就绪后再切换。

#### 6.1.5 归档周期

1. 点击 **「归档」** 按钮。
2. 在弹出的确认框中点击 **「确定」**。
3. 归档后数据将被锁定，不可再编辑。

本页面包含 **5 个选项卡**，分别配置不同类型的参数。

**操作按钮：**
- **「+ 添加行」**：新增一行部门/组别配置。
- **「保存」**：保存所有修改。

#### 6.2.2 专项目标值（选项卡 2）

配置全局专项目标：

| 参数 | 说明 |
|------|------|

| 科技创新目标 | 科技创新目标金额 |

编辑后点击 **「保存」**。

#### 6.2.3 项目类型系数（选项卡 3）

配置各项目类型对应的系数：

| 项目类型 | 系数 |
|----------|------|
| 集成 | 可编辑 |
| 综合 | 可编辑 |
| 自研 | 可编辑 |
| … | … |

**操作按钮：**
- **「+ 添加」**：新增项目类型。
- **「删除」**：删除某一行。
- **「重置默认值」**：恢复系统预设系数。
- **「保存」**：保存修改。

#### 6.2.4 员工指标系数（选项卡 4）

配置各职级对应的指标系数（如 T1~T9、S1~S5、P1~P5 等）。

操作方式与项目类型系数一致：支持添加、删除、重置默认值和保存。

#### 6.2.5 签约概率设置（选项卡 5）

与项目管理页面的签约概率设置功能相同，列出未签约项目，可编辑概率并保存。

## 七、填报申报

### 7.1 项目参与度

**菜单路径：** 填报申报 → 项目参与度  
**适用角色：** 管理员、项目经理

#### 7.1.1 项目经理操作

项目经理只能查看和编辑**自己负责的项目**。

**填写参与度：**
1. 在左侧面板选择一个项目。
2. 右侧显示该项目的参与人员列表及参与系数。
3. 点击 **「+ 添加员工」** 向项目中添加参与人员。
4. 在每位员工旁的输入框中填写参与系数（0 ~ 1）。
5. 确认填写正确后，点击 **「保存」** 暂存数据，或点击 **「提交」** 正式提交。

## 十二、完整考核流程指南

以下是一个完整的考核周期操作流程，按角色和阶段组织：

### 阶段 1：数据导入（管理员）

```
1. 考核设置 → 考核周期 → 创建新周期 → 激活
2. 基础数据 → 员工管理 → 导入/更新员工数据
3. 基础数据 → 项目管理 → 导入/更新项目数据
4. 考核设置 → 考核参数 → 配置各项参数
   - 部门人均目标值
   - 专项目标值
   - 项目类型系数
   - 员工指标系数
   - 签约概率
5. 确认无误后 → 切换至"阶段 2：填报申报"
```

### 阶段 2：填报申报（项目经理 + 员工 + 管理员）

```
项目经理：
1. 填报申报 → 项目参与度 → 为所负责项目填报参与度 → 提交

所有员工（含项目经理和领导）：
2. 填报申报 → 公共积分申报 → 申报公共/转型活动

管理员：
3. 填报申报 → 项目参与度 → 审核各项目参与度填报情况
4. 填报申报 → 公共积分申报 → 审核并调整积分
5. 确认填报完成后 → 切换至"阶段 3：在线评分"
```

### 阶段 3：在线评分（全员 + 领导 + 管理员）

```
管理员：
1. 360 评价 → 互评关系 → 生成互评关系
2. 检查并调整互评关系分配

全员：
3. 360 评价 → 我的评价 → 逐一完成所有评价任务

领导：
4. 360 评价 → 工作目标 → 为公共人员评分

管理员：
5. 监控评价完成进度（工作台统计卡片）
6. 确认所有评价完成后 → 切换至"阶段 4：结果查看"
```

### 阶段 4：结果查看（管理员 + 领导）

```
管理员：
1. 统计分析 → 积分统计 → 触发计算 → 审核明细和汇总
2. 统计分析 → 经济指标 → 触发计算 → 审核数据
3. 结果管理 → 加减分/重点任务 → 录入加减分记录和重点任务分数
4. 结果管理 → 最终成绩 → 触发计算
5. 结果管理 → 最终成绩 → 设置评定等级

领导 + 管理员：
6. 结果管理 → 最终成绩 → 编辑领导评语

管理员：
7. 导出报表 → 分发给相关人员审阅
8. 确认无误后 → 切换至"阶段 5：确认结束"
```

### 阶段 5：确认结束（管理员）

```
管理员：
1. 最终确认所有数据正确
2. 结果管理 → 最终成绩 → 确认考核完成
3. 系统归档当前周期，数据永久锁定
```

A：请确认输入的是 11 位以 1 开头的手机号。

### B.2 填报相关

**Q：项目参与度提交后还能修改吗？**  
A：可以。在填报申报阶段内，已提交的参与度仍可重新编辑和提交。

**Q：参与度校验不通过怎么办？**  
A：系统要求同一部门内所有参与员工的系数之和为 1.0（允许 ±0.01 误差）。请检查并调整各员工的系数。

**Q：公共积分的分值是如何计算的？**  
A：系统根据活动类型、人月数和复杂度自动计算规模值和复杂度值，再乘以工作量系数得出最终积分。管理员可手动覆盖计算结果。

### B.3 评价相关

**Q：360 评价提交后可以修改吗？**  
A：不可以。评分一经提交即被锁定。管理员可通过后台重置特定评价关系的评分。

**Q：为什么我看不到评价任务？**  
A：请确认当前周期已进入"在线评分"阶段，并且管理员已生成互评关系。

### B.4 成绩相关

**Q：最终成绩是如何计算的？**  
A：最终成绩 = 工作积分得分 + 经济指标得分 + 360 综合评价得分 + 加减分 + 重点任务分数（基层管理人员）+ 工作目标完成度分数（公共人员）。具有双重考核类型的员工按 50/50 权重合并。

**Q：评定等级有哪几种？**  
A：共 4 个等级 — 优秀、合格、基本合格、不合格。由管理员在最终成绩页面手动设定。

**Q：确认考核完成后还能修改数据吗？**  
A：不能。确认考核完成后，当前周期所有数据将被永久锁定和归档。此操作不可撤销。

### B.5 系统相关

**Q：如何切换深色/浅色模式？**  
A：点击顶部导航栏右侧的主题切换图标，可选择"跟随系统"、"浅色模式"或"深色模式"。

**Q：系统支持哪些浏览器？**  
A：推荐使用 Chrome、Edge、Firefox 等现代浏览器的最新版本。
